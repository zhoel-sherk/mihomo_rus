# Примеры конфигурации Mihomo — для кого эта папка

Если вы **впервые** сталкиваетесь с Mihomo (Clash Meta), начните не отсюда, а с корневого [**`GUIDE.md`**](../../GUIDE.md): там объясняется, что такое `config.yaml`, режимы `mode`, блоки `dns:`, `rules:`, `proxies:` и типичные ошибки.

Эта папка — **дополнение**: отдельные файлы показывают *одну тему глубже*, с комментариями прямо в YAML. Они рассчитаны на тех, кто уже понимает идею «один текстовый файл настроек», но хочет увидеть **связку нескольких блоков** (DNS + sniffer, TUN + DNS, провайдеры + группы и т.д.).

---

## Минимум терминов

| Термин | Простыми словами |
|--------|------------------|
| **Mihomo** | Прокси-ядро: принимает трафик, по правилам решает «напрямую / через какой узел / отбросить», может подменять DNS и маршрут. |
| **`config.yaml`** | Главный файл настроек. Обычно лежит в **каталоге данных** ядра (параметр запуска **`-d`**; на Entware часто `/opt/etc/mihomo/`). Имя на диске почти всегда `config.yaml`. |
| **YAML** | Формат «отступами важен»: вложенность только **пробелами**, табы лучше не использовать. Один лишний пробел — и ядро напишет `Parse config error`. |
| **`mode: rule`** | Основной режим: решение «куда пойдёт соединение» берётся из списка **`rules:`** сверху вниз до первого совпадения. |
| **`proxies:`** | Список **исходящих узлов** (VLESS, Shadowsocks, …). Имена из этого списка потом используют в **`proxy-groups:`** и в правилах. |
| **`proxy-groups:`** | Логические «ручки»: пользователь или авто-логика выбирают узел (группа `select`, `url-test`, …). В **`rules:`** указывают **имя группы**, а не имя отдельного прокси (если не нужно иное). |
| **`rules:`** | Условия маршрутизации: домен, IP, страна, внешний список (`RULE-SET`), … и куда отправить трафик (`DIRECT`, `REJECT`, имя группы). |
| **`listeners:`** | **Входящие** порты на устройстве с Mihomo (SOCKS, HTTP, mixed…). Отличается от «классических» `mixed-port` / `socks-port` в корне — см. гайд. |
| **`dns:`** | Встроенный DNS Mihomo: как резолвить имена для правил и приложений, режим **fake-ip** vs **redir-host**, отдельные серверы для разных зон. |

Подробно по типам узлов: [**`docs/ru/proxies.md`**](../../docs/ru/proxies.md). Панели и API: [**`docs/ru/web-ui.md`**](../../docs/ru/web-ui.md).

## Чем `examples/` отличается от `mihomo/config.yaml`

| | **[`../config.yaml`](../config.yaml)** | **`examples/scenario-*.yaml`** |
|---|----------------------------------------|--------------------------------|
| Назначение | Один **цельный** пример «как может выглядеть рабочий роутер» без секретов. | **Учебные срезы**: одна тема, больше пояснений в комментариях. |
| Можно ли копировать целиком на роутер? | Да, после подстановки своих прокси и проверки путей. | Только после **слияния** с вашим конфигом и удаления дубликатов ключей. |
| `proxies:` | Пустой массив — вы добавляете узлы. | То же; в сценарии 03 есть заглушка `inline` для демонстрации синтаксиса. |

## Как безопасно перенести кусок в свой `config.yaml`

1. Откройте свой текущий `config.yaml` и **сделайте копию** (бэкап).
2. Откройте нужный `scenario-*.yaml` и найдите блок (например только `dns:` или только `tun:`).
3. **Вставьте** этот блок в свой файл на том же уровне отступов, что и соседние корневые ключи (`dns:`, `rules:`, …).
4. Убедитесь, что **нет двух одинаковых** корневых ключей (двух `dns:` или двух `tun:` — YAML это не объединит автоматически; нужно руками слить содержимое в один блок).
5. Перезапустите Mihomo и посмотрите лог. При ошибке парсинга ядро обычно укажет строку — правьте отступы и кавычки.

Проверка синтаксиса (если у вас есть бинарник Mihomo): `mihomo -t -d /путь/к/каталогу` — читает `config.yaml` из этого каталога.

|------|----------|----------------------|
| [scenario-01-dns-policies.yaml](scenario-01-dns-policies.yaml) | Хотите разобраться, **почему** у вас «ломается сайт» или DNS у провайдера | `default-nameserver` vs `nameserver`, `fallback`, `nameserver-policy` + `geosite:`, **fake-ip**, `fake-ip-filter`, связка со **sniffer** |
| [scenario-02-tun-dns-hijack.yaml](scenario-02-tun-dns-hijack.yaml) | Нужен **виртуальный сетевой адаптер** (TUN), «весь трафик через Mihomo» на ПК/Linux | `tun:`, `dns-hijack`, `routing-mark`; предупреждение про **Keenetic + nfqws** |
| [scenario-03-proxy-providers-url-test.yaml](scenario-03-proxy-providers-url-test.yaml) | Есть **ссылка на подписку** или свой файл со списком узлов | `proxy-providers` (http / file / inline), группы **`url-test`**, **`fallback`**, **`load-balance`**, healthcheck |
| [scenario-04-sub-rules-tunnels-ntp.yaml](scenario-04-sub-rules-tunnels-ntp.yaml) | Хотите **разгрузить** длинный `rules:` или отдельную логику на порт / туннель | `sub-rules:`, вызов **`SUB-RULE`**, `listeners` с полем **`rule:`**, **`tunnels:`**, **`ntp:`**, **`experimental:`** |
| [scenario-05-listeners-auth.yaml](scenario-05-listeners-auth.yaml) | Нужно **несколько входящих портов** и пароль для LAN | Несколько **`listeners:`**, глобальные **`authentication`** / **`skip-auth-prefixes`** |
| [scenario-06-zkeenip-rulesets.yaml](scenario-06-zkeenip-rulesets.yaml) | Нужны **готовые .mrs** с IP CDN/хостингов (zkeen-ip) под `RULE-SET` | Якорь YAML, `rule-providers` (ipcidr + releases URL), пример `rules:` — полный каталог в [zkeenip-rulesets](https://github.com/zxc-rv/zkeenip-rulesets) |
| [scenario-07-ad-filter-mrs.yaml](scenario-07-ad-filter-mrs.yaml) | **Реклама**: один доменный список в `.mrs` (Hagezi Pro) | `adlist` + `RULE-SET,adlist,REJECT` — см. [ad-filter](https://github.com/zxc-rv/ad-filter) |

**Сторонние rule-set** (не часть ядра Mihomo): полный список URL и имён провайдеров всегда сверяйте с README upstream-репозитория; файлы в `examples/` — укороченные шаблоны.

## В каком порядке читать

1. [**`GUIDE.md`**](../../GUIDE.md) — оглавление слева вверху; минимум разделы про **`mode`**, **`dns:`**, **`rules:`**, **`proxy-groups:`**.
2. Затем откройте **`../config.yaml`** — как выглядит «всё вместе» на одном экране.
3. Потом зайдите в **один** сценарий под вашу задачу и читайте **комментарии сверху файла** — там повтор ключевых идей простыми словами.

Если на роутере рядом стоят Xray и Mihomo и вы путаетесь в файлах `.dat` / `GeoSite.dat`, см. [**`docs/ru/geo-xray-dat.md`**](../../docs/ru/geo-xray-dat.md).

## Важно про секреты и законность

В примерах встречаются **выдуманные** URL подписок и пароли. Не вставляйте в публичные репозитории свои **реальные** ссылки на подписки, `uuid`, ключи и IP. Соответствие законам страны и правилам провайдера — ваша ответственность; документы здесь только про **техническую** настройку ядра.
